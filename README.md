## 🧠 Model

| Detail | Value |
|--------|-------|
| Task | Binary classification (churn / no churn) |
| Best model (typical) | Gradient Boosting |
| **Calibration** | `CalibratedClassifierCV` with isotonic regression (fixes overconfident raw probabilities) |
| Threshold | Optimised for ≥ 70% recall on validation set |
| Tracking | MLflow experiment runs |

### Why Probability Calibration?
Tree-based models (Random Forest, Gradient Boosting) often produce poorly calibrated probabilities — they tend to be overconfident near 0 and 1. We wrap the best model in `CalibratedClassifierCV(method='isotonic', cv=5)` so that a predicted probability of 0.70 actually means 70% of those customers churn. The **Brier Score** in the metadata confirms calibration quality.

## 🚀 Quickstart (Local)

### 1. Clone the repository

```bash
git clone https://github.com/<your-org>/churn-mlops.git
cd churn-mlops
```

### 2. Install dependencies

```bash
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Train the model

```bash
python train.py
# Optional flags:
#   --data  path/to/custom.csv
#   --experiment  my-experiment-name
```

Artifacts saved to `models/`:
- `churn_model_latest.joblib`  – calibrated classifier
- `preprocessor.joblib`        – fitted ColumnTransformer
- `metadata.json`              – metrics, threshold, feature list

View MLflow UI:
```bash
mlflow ui
# Open http://localhost:5000
```

### 4. Run the API

```bash
uvicorn app:app --reload
# API docs: http://localhost:8000/docs
```

**Predict single customer:**
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "gender": "Female", "SeniorCitizen": 0, "Partner": "Yes",
    "Dependents": "No", "tenure": 12, "PhoneService": "Yes",
    "MultipleLines": "No", "InternetService": "DSL",
    "OnlineSecurity": "No", "OnlineBackup": "Yes",
    "DeviceProtection": "No", "TechSupport": "No",
    "StreamingTV": "No", "StreamingMovies": "No",
    "Contract": "Month-to-month", "PaperlessBilling": "Yes",
    "PaymentMethod": "Electronic check",
    "MonthlyCharges": 29.85, "TotalCharges": 358.20
  }'
```

**Response:**
```json
{
  "churn": true,
  "churn_probability": 0.7231,
  "no_churn_probability": 0.2769,
  "threshold_used": 0.35,
  "risk_label": "High"
}
```

**Batch prediction (CSV upload):**
```bash
curl -X POST http://localhost:8000/predict/batch \
  -F "file=@data/synthetic/synthetic_customers.csv"
```

### 5. Run the Streamlit dashboard

```bash
streamlit run streamlit_app.py
# Opens http://localhost:8501
```

### 6. Generate synthetic test data

```bash
python generate_synthetic_data.py --n 500 --output data/synthetic/test_customers.csv
# Options:
#   --n           number of customers (default: 500)
#   --churn_rate  approximate churn rate (default: 0.265)
#   --seed        random seed for reproducibility
```

## ⚙️ CI/CD Pipeline

Every push to `main` or `develop` triggers the GitHub Actions workflow:

```
lint (ruff) → unit tests (pytest + coverage) → train model → validate metrics → build Docker → deploy to Render
```

| Step | Tool | Description |
|------|------|-------------|
| Lint | ruff | PEP 8 + import checks |
| Test | pytest | Unit tests with ≥ 60% coverage gate |
| Train | scikit-learn + MLflow | Trains on full dataset; ROC-AUC gate ≥ 0.75 |
| Build | Docker Buildx | Multi-platform image build |
| Deploy | Render webhook | Auto-deploys on `main` push |

### Setting up CI/CD deploy to Render

1. Create a [Render.com](https://render.com) account and deploy from GitHub.
2. In Render, go to your service → **Settings** → **Deploy Hook** → copy the URL.
3. In GitHub → **Settings** → **Secrets and variables** → **Actions** → add secret `RENDER_DEPLOY_HOOK_URL`.

3. **Hard-coded relative paths** – replaced with `Path` objects resolved from project root.
4. **No error handling in preprocessing** – added column validation and `TotalCharges` type coercion in the API layer.
5. **German print statements** – replaced with English for consistency (`Ordner erstellt` → proper logging).
